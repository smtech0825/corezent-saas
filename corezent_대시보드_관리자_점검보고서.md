## 한눈 요약 — 가장 시급한 문제 3가지

**1. 옛 제품 6종이 관리자에서 모두 "활성" 상태라, 프론트의 옛 제품 노출이 여기서 흘러나온다. (근본 원인이 관리자에 있음)**
관리자 → 제품 화면에 GenieWork 1PC 외에 GenieStock(lite/Pro/Max)·GeniePost(/Pro/Max) 6종이 전부 **활성**으로 등록돼 있습니다. 공개 사이트가 옛 제품으로 도배되는 이유가 이것입니다. 또 공지 배너 편집기에는 "GeniePost 출시 — AI 워드프레스 자동 포스팅, 월 $6.99부터"가 그대로 입력돼 있습니다. **다행히 관리자에 이를 끄거나 고칠 도구(제품 활성/비활성, 공지 배너, 프론트엔드 CMS)가 이미 있습니다.**

**2. 사용자용 "제휴(추천 리워드) 프로그램"이 공무원 고객에게 부적절하다. (이해충돌·청탁금지법 리스크)**
대시보드에 친구를 추천 링크로 초대하면 결제 확정 후 "스토어 크레딧"을 적립해주는 제휴 프로그램이 켜져 있습니다(최소 출금 $50.00). 고객의 99%가 공무원인데, 동료를 소개하고 금전적 보상을 받는 구조는 **이해충돌·청탁금지법(김영란법) 측면에서 문제가 될 수 있습니다.** 공무원 대상 서비스에서는 숨기는 것을 강력히 권합니다.

**3. 회원 화면 전반이 USD·다크모드이고, 기관 구매에 필요한 증빙·정보 입력이 없다.**
대시보드 결제 내역·구독·관리자 매출이 모두 `$0.99 / $11.99` 등 USD로 표기됩니다. 결제 내역에 **세금계산서·영수증 다운로드 버튼이 없고**, 설정에는 **기관 정보(기관명·사업자번호 등) 입력란이나 회원 탈퇴·알림 설정이 없습니다.** 공공기관 구매·정산에 필요한 최소 요소가 빠져 있습니다.

# A. 사용자 대시보드(/dashboard)

## A-1. 개요(/dashboard)  🟡 중간

**현재 상태**
- 상단 통계 카드: 전체 라이선스 12, 활성 구독 3, 전체 주문 5.
- "활성 구독" 목록·"최근 주문" 목록에 GenieWork 1PC가 표시되며, 금액은 모두 `$0.99`(USD).
- 다크모드 기본.

**문제점**
- 금액이 USD로 표기되어 한국 사용자에게 직관적이지 않음.
- 다크모드 단일 테마(라이트 선택지 없음).

**개선 방향**
- 금액 표기를 KRW로 전환(시스템 통화는 LemonSqueezy/제품 가격 정책과 함께 정리). 최소한 회원 화면에서는 원화 환산 병기.
- 라이트 모드 지원(공공기관 사용자 가독성·인쇄 고려).

**우선순위:** 🟡 중간

## A-2. 라이선스(/dashboard/licenses)  🔴 높음

**현재 상태**
- 라이선스 키 목록(총 12개): 라이선스 키(복사), 제품, 상태(활성/해지), 주기(월간), 만료일. 페이지네이션 1/2.
- 제품 칸에 GenieWork 1PC뿐 아니라 GenieStock Max·GeniePost 등 옛 제품 키가 섞여 있음(테스트 데이터).
- 행마다 **"PC 교체/재인증(HWID 초기화)" 같은 셀프 액션 버튼이 보이지 않음**(복사 외에 없음).

**문제점**
- GenieWork는 HWID(기기) 바인딩 제품이라, 사용자가 PC를 교체하면 라이선스 재인증(reset)이 필요함. 그런데 사용자가 **스스로 PC 교체(재인증)를 할 UI가 없어 보임** → 문의 폭증·이탈 위험.
- 옛 제품 라이선스가 목록에 노출(실데이터에선 혼선).

**개선 방향**
- 라이선스 행에 **"PC 교체(재인증)" 버튼**을 추가해, 사용자가 직접 기존 기기 바인딩을 해제(reset)하고 새 PC에서 활성화할 수 있게 안내(`/api/license/reset` 연동). 횟수 제한·안내 문구 포함.
- 라이선스 화면에 "현재 인증된 PC(기기명/HWID 일부)" 표시로 사용자 이해도 향상.
- 운영 데이터에서 옛 제품 라이선스 정리(테스트 데이터 분리).

**우선순위:** 🔴 높음

## A-3. 결제(/dashboard/billing)  🔴 높음

**현재 상태**
- "구독(총 14개)": GenieWork 1PC 항목별 상태(활성/만료됨)와 "라이선스 확인", "구독 취소" 버튼.
- "결제 내역(총 14개)": 제품/날짜/금액/상태 표. 금액 `$0.99`(USD), 상태 한글(결제 완료/환불됨).
- **세금계산서·영수증 다운로드 또는 고객 포털(결제수단 변경/영수증) 링크가 보이지 않음.**

**문제점**
- 기관 구매 시 필수인 **세금계산서·현금영수증·영수증 발급 경로가 없음.**
- 금액 USD 표기.
- (구조적) LemonSqueezy는 Merchant of Record라 LS 명의 인보이스만 가능 → (주)에스엠테크 명의 한국 세금계산서를 회원이 받을 방법이 화면에 전혀 없음.

**개선 방향**
- 결제 내역 각 건에 **영수증(인보이스) 보기/다운로드** 추가(LS 영수증 링크라도 우선 연결).
- **세금계산서·현금영수증 발행 요청** 기능 또는 안내(별도 발행 프로세스) 추가 — 기관 구매 대응의 핵심.
- LemonSqueezy 고객 포털(Signed URL)을 "결제수단 변경/영수증" 버튼으로 연결.
- 금액 KRW 표기(환산 병기).

**우선순위:** 🔴 높음

## A-4. 제휴(/dashboard/affiliate)  🔴 높음 — 공무원 적합성

**현재 상태**
- "제휴 프로그램": 추천 링크로 친구 초대 → 결제 확정 후 스토어 크레딧 적립. 추천 코드/링크(예: corezent.com/r/XXXX) 제공.
- 통계: 클릭/가입/전환 0, 스토어 크레딧 잔액 `$0.00`, "최소 진환 금액 $50.00 · 다음 결제 시 할인으로 사용".
- 문구 오타: **"진환"**(→ "전환" 또는 "출금/정산"이 맞음)이 통계 카드·안내문에 반복됨.

**문제점**
- 동료 추천에 금전적 보상을 주는 리워드 구조는 **공무원 대상 서비스에 부적절**(이해충돌·청탁금지법 우려). 옛 B2C(블로그·주식) 제품의 잔재.
- 금액 USD(`$0.00 / $50.00`).
- "진환" 오타로 신뢰도 저하.

**개선 방향**
- **공무원 타깃에서는 제휴(추천 리워드) 메뉴를 숨김/비활성화**하는 것을 권장. 유지가 필요하면 보상을 크레딧이 아닌 비금전 형태로 재설계하고 약관에 공직자 관련 유의사항 명시.
- 유지 시: 금액 KRW 전환, "진환" → "전환/정산" 오타 수정.

**우선순위:** 🔴 높음

# B. 관리자 콘솔(/admin)

## B-1. 개요(/admin)  🟡 중간

**현재 상태**
- 통계 카드: 총 사용자 4, 총 매출 `$15.95`, 활성 라이선스 5, 열린 티켓 0.
- "최근 주문" 표: 금액 USD, 상태가 **영문 "Paid / Refunded"** (한글 UI와 혼재).
- "최근 가입자": 이승용(User), CoreZent(Admin), Steve Yoo(User) 등.

**문제점**

**개선 방향**
- 매출/금액 KRW 표기(또는 원화 병기).

## B-2. 제품(/admin/products)  🔴 높음 — 가장 중요

**현재 상태**
- 7개 제품이 **전부 활성**:
  1. GenieWork 1PC — `$0.99/mo`, 연간 "—"(없음), 설명/카테고리 빈약(카테고리 desktop)
  2. GenieStock lite — `$6.99/mo · $69/yr`
  3. GenieStock Pro — `$11.99/mo · $119/yr`
  4. GenieStock Max — `$6.99/mo · $159/yr` ← **월가가 Pro보다 쌈(데이터 오류)**
  5. GeniePost — `$6.99/mo · $69/yr`
  6. GeniePost Pro — `$11.99/mo · $119/yr`
  7. GeniePost Max — `$15.99/mo · $159/yr`
- 카테고리는 전부 "desktop". 제품 추가/수정/삭제·순서 변경(화살표) 가능.

**문제점**
- **옛 제품 6종이 활성이라 공개 사이트(메인·요금제)에 그대로 노출됨 = 프론트 문제의 근본 원인.**
- GenieWork만 설명·연간 요금·등급(라이트/프로 등) 구성이 비어 있음 → 정작 주력 제품이 가장 부실.
- 가격 USD, 가격 데이터 비일관(Max < Pro), 카테고리 미분류.

**개선 방향**
- **GenieStock·GeniePost 6종을 비활성화(또는 삭제).** 공개 노출을 GenieWork로 한정.
- GenieWork 1PC의 설명·기능·연간 요금·(필요 시) 다중 PC/기관용 등급을 채움.
- 가격 KRW 전환, 가격 정합성 점검, 카테고리를 행정/공공 맥락으로 재정의.

**우선순위:** 🔴 높음

## B-3. 라이선스(/admin/licenses)  🟡 중간

**현재 상태**
- 총 12개, 상태 필터·이메일 검색 제공. 컬럼: 시리얼 키, 제품, 사용자, 상태(활성/취소됨), 결제 주기, 갱신일, 만료일, 작업.
- 활성 라이선스에 **"회수(Revoke)"** 버튼 있음. 옛 제품(GenieStock Max·GeniePost) 라이선스도 목록에 존재.

**문제점**
- 관리자에 **HWID(기기) 재설정/초기화 액션이 보이지 않음**(회수만 있음). PC 교체 문의를 관리자가 직접 처리하기 어려울 수 있음.
- 옛 제품 라이선스 혼재(테스트 데이터).

**개선 방향**
- 라이선스 상세에 **HWID 초기화(reset)** 액션 추가(사용자 PC 교체 대응). GenieWork는 별도 Supabase(`hwid_mapping`) 연동.
- 운영 시 옛 제품 라이선스 정리.

**우선순위:** 🟡 중간

## B-4. 제휴(/admin/affiliate)  🟡 중간

**현재 상태**
- 관리자에 제휴(추천 리워드) 관리 메뉴 존재(사용자용 제휴 프로그램과 짝).

**문제점**
- A-4와 동일 맥락 — 공무원 대상엔 부적절한 기능. 운영하지 않을 거면 관리 메뉴도 정리 대상.

**개선 방향**
- 제휴 프로그램을 접는다면 사용자/관리자 양쪽 메뉴를 함께 숨김 처리.

**우선순위:** 🟡 중간

## B-5. 프론트엔드 CMS(/admin/content/*)  🔴 높음

**현재 상태**
- 하위 편집기: **공지 배너 / 섹션 설정 / 히어로 / 소개 / 이용 방법 / 특징 / 고객 후기**(FAQ 등 추가).
- 공지 배너 편집기(/admin/content/announcement): 배너 표시(표시/숨김), 배너 텍스트(데스크톱/모바일)에 **"GeniePost 출시 — AI 워드프레스 자동 포스팅, 월 $6.99부터"**, 링크 텍스트 "more →", 링크 URL `/product`, 저장.

**좋은 점(중요)**
- 공개 사이트의 배너·히어로·후기·섹션 노출/순서를 **코드 수정 없이 여기서 바꿀 수 있음.** 즉 앞 보고서에서 지적한 프론트 문제(옛 배너·막연한 히어로·외국인 후기)를 **관리자에서 직접 교체 가능.**

**문제점**
- 현재 CMS 콘텐츠가 옛 제품 기준으로 채워져 있음(공지 배너가 대표 사례).

**개선 방향(구체적으로)**
- **공지 배너**: 텍스트를 GenieWork 공지로 교체하거나 "숨김". 링크 URL도 GenieWork로.
- **히어로**: "공무원·공공기관용 AI 행정 문서 자동화 — 야근↓·감사 걱정↓·개인정보는 PC 안에서" 류로 교체.
- **고객 후기**: 외국인 블로거 후기 → 공무원/기관 사례(익명)로 교체.
- **섹션 설정**: 옛 제품 위주 섹션 visibility/순서 재정렬.
- **소개·이용 방법·특징**: GenieWork(다운로드→설치→라이선스 인증) 기준으로 재작성.

**우선순위:** 🔴 높음

## B-6. 시스템 설정(/admin/settings)  🔴 높음 — 법적 표시

**현재 상태**
- **일반 설정**: 사이트 이름(CoreZent), 사이트 URL(https://corezent.com), 고객지원 이메일(support@corezent.com), 푸터 저작권.
- **푸터 정보**: 자유 입력 텍스트(줄바꿈 그대로 반영). 현재 "회사명: CoreZent by SMTech co,.Ltd / 사업자등록번호: 357-88-01674 …".
- **SEO 설정**: GA 추적 ID, 메타 제목/설명/키워드.
- **SMTP 설정**: 메일 발송 구성.
- **결제 통화 설정은 없음**(통화는 LemonSqueezy 스토어 설정에서 관리).

**문제점**
- 푸터 정보에 **통신판매업 신고번호·사업장 주소·고객센터 전화번호가 빠져 있음**(전자상거래법 표시 의무 미충족) — 단, 이 칸이 자유 입력이라 **여기서 바로 보완 가능.**
- 통화(USD→KRW) 전환은 이 화면에서 불가 → LemonSqueezy 대시보드에서 스토어 통화 변경 + 제품 가격 재설정 필요(구조적).

**개선 방향**
- 푸터 정보에 **통신판매업 신고번호, 사업장 주소, 고객센터 전화/이메일** 줄을 추가해 전자상거래법 충족.
- SEO 메타(제목·설명·키워드)를 GenieWork·공무원 키워드로 채움. 회원 페이지 개별 타이틀 누락은 코드 측 metadata로 보완.
- 통화 전환은 LemonSqueezy 스토어 설정에서 KRW로 변경 + 제품 가격 KRW 재입력(관리자 제품 화면과 함께).

**우선순위:** 🔴 높음

## 종합 개선 우선순위

**1순위 (즉시 — 제품 정체성·법적 필수)**
- 관리자 제품에서 GenieStock·GeniePost 6종 **비활성화**, GenieWork 정보 채우기.
- 프론트엔드 CMS에서 **공지 배너·히어로·고객 후기**를 GenieWork(공무원) 기준으로 교체.
- 시스템 설정 푸터 정보에 **통신판매업 신고번호·주소·연락처** 추가.

**2순위 (결제·기관 구매 대응)**
- 통화 KRW 전환(LemonSqueezy 스토어 + 제품 가격 재설정) 및 회원/관리자 금액 표기 통일.
- 대시보드 결제에 **영수증·세금계산서/현금영수증 발행** 경로 추가.
- 설정에 **기관 정보 입력란**(기관명·사업자번호·세금계산서 수신처) 신설.

**3순위 (공무원 적합성·기능 보강)**
- 사용자 **제휴(추천 리워드) 프로그램 숨김**(청탁금지법 리스크) — 사용자/관리자 메뉴 함께.
- 라이선스에 **PC 교체(HWID 재설정)** 기능 추가(사용자 셀프 + 관리자 양쪽).
- 회원 설정에 탈퇴·알림 설정 추가.

**4순위 (정합성·디테일)**
- 주문 상태 영문→한글("결제완료/환불됨"), "진환"→"전환/정산" 오타 수정.
- 회원 페이지 전용 metadata(title) 지정(설정·고객지원 등).
- 라이트 모드 지원, 가격 데이터 비일관(Max<Pro) 수정.
