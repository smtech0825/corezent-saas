# audit-light-mode-public

You are a senior Next.js 15 (App Router) / TypeScript / Tailwind v4 engineer on the **CoreZent** web codebase. **READ-ONLY 점검.** 공개 사이트(랜딩·요금제 등)에 **라이트 모드를 기본값**으로 하고 **토글로 다크 전환** 가능하게 만들기 위한 사전 점검이다. 코드/커밋/푸시 변경 절대 금지 — 현황과 작업 방식만 `파일:줄` 근거로 보고.

## 목표 (이번엔 점검만)
- 공개 사이트 = 랜딩(/), 요금제(/pricing), 제품(/product), 그 외 비로그인 공개 페이지(about/contact/faq/changelog/legal 등) + 그 페이지들이 쓰는 components/sections·Navbar·Footer.
- **제외**: 회원 대시보드(/dashboard)·관리자(/admin)는 이번 범위 아님. 단, 공통 컴포넌트(Navbar/Footer 등)가 공개·회원 양쪽에서 쓰이면 그 사실을 명시(영향 범위 파악용).
- 원하는 최종: 라이트 기본 + 토글로 다크. 지금은 그게 **얼마나 큰 작업인지**를 결정하는 점검.

## 점검 항목 (전부 READ-ONLY · 파일:줄 근거)

### 1. 현재 색상 시스템 구조
- globals.css의 `@theme`/CSS 변수 정의 확인: 색상이 **토큰(변수)로 정의**돼 있는지, 라이트/다크 변수 세트가 이미 있는지.
- Tailwind v4 설정에서 다크모드 전략: `dark:` variant를 쓰는가? `class` 기반인가 `media`(prefers-color-scheme) 기반인가? 현재 기본이 다크로 어떻게 고정돼 있는지(html에 클래스? CSS 변수 기본값? media query?).
- 결론: **색상이 토큰화돼 있나, 하드코딩(hex/임의값)인가** — 이게 작업량의 핵심.

### 2. 하드코딩 색상 실태 (작업량 추정의 핵심)
- 공개 사이트 파일들에서 **하드코딩 색상** 사용을 grep으로 집계: `bg-[#...]`, `text-[#...]`, 인라인 style 색상, `#xxxxxx` 리터럴, `rgb(...)` 등.
- 토큰(변수/Tailwind 시맨틱 클래스 `bg-background` 등) 사용 vs 하드코딩 사용 **비율·파일 목록**.
- 특히 다크 전제로 박힌 색(어두운 배경 hex, 밝은 텍스트 hex)이 어디에 몇 곳 있는지 — 라이트로 안 바뀌면 깨지는 지점들.

### 3. 다크 전제 시각 요소
- 그라데이션·글로우·글래스(반투명) 효과가 다크 배경 전제로 만들어진 곳(랜딩 Hero 등) — 라이트에서 그대로 두면 안 보이거나 깨지는 것들. 파일:줄.
- 이미지/SVG/로고가 다크 배경 전제(흰색 로고 등)라 라이트에서 안 보이는 것.

### 4. 토글 구현 방식 후보
- 토글 상태를 어디에 저장할지: localStorage(주의: 일부 환경 제약)·쿠키·서버. SSR(page.tsx force-dynamic)과의 상호작용, 초기 깜빡임(FOUC) 방지 가능 여부.
- 토글 UI를 어디 둘지(Navbar 등).
- ⚠️ 추측 말고, 현재 코드 구조에서 **가장 단순하고 FOUC 없는 방식**을 제안.

### 5. 작업량·위험 판정 (결론)
- 다음 중 어디에 해당하는지 판정:
  - (A) **토큰화돼 있음** → 라이트/다크 변수 두 벌 + 토글만. 적음.
  - (B) **부분 토큰화** → 토큰 보강 + 하드코딩 일부 교체. 중간.
  - (C) **하드코딩 많음** → 공개 사이트 색상을 토큰으로 마이그레이션 선행 필요. 큼.
- 각 경우의 예상 작업 Wave 구성 제안(점검 결과 기반, 실행은 승인 후 별도).
- 회귀 위험: 공통 컴포넌트가 회원/관리자와 공유되면 공개만 바꿀 때 그쪽에 영향 가는지.

## 출력 형식
- 위 1~5를 `파일:줄` 근거로. 색상 하드코딩은 집계 수치(파일 수·발생 수)와 대표 위치.
- 마지막에 **판정(A/B/C)**과 **권장 작업 방식 1개**(Wave 구성)를 명시. 코드는 손대지 말 것.
