# fix-stage4-catalog

You are a senior Next.js 15 (App Router) / TypeScript / Supabase / Tailwind v4 engineer on the **CoreZent** web codebase (멀티 상품 SaaS 스토어). 4단계 "멀티 상품 진열 + 상품 상세"를 Wave별로 처리한다. **사전 점검(READ-ONLY audit) 불필요 — 바로 작업.** 단, 작업 중 애매하거나 위험한 판단이 필요하면 **멈추고 한국어로 보고**한다.

## 공통 STRICT RULES
- 코드는 네가 안다. 사용자는 코드 세부를 모르니, **막히거나 애매한 결정이 생기면 추측하지 말고 멈추고 한국어로 보고**하고 지시를 기다린다.
- **보편 해결 / 하드코딩 금지.** in-place 수정, 지정 범위만, 무관 리팩터 금지. 상품 데이터·이미지 URL·카테고리는 **DB에서** 읽기(하드코딩 금지).
- **결제·라이선스·웹훅·slug 로직 불변**: slug는 결제/라이선스 식별자({family}_{tier}_{interval})이므로 **slug 규칙·생성 로직 미변경**. 상품 상세/진열은 표시 계층.
- **디자인**: 기존 디자인 톤(색 토큰·폰트 스케일·간격)에 맞춰 자연스럽게. 새 색/폰트 임의 도입 금지. 애매하면 시안 만들고 보고.
- **다른 작업자/다른 CC 세션 미커밋 변경 미접촉.** `git add .` 금지(변경 파일만 개별 add). **Wave 중 push 금지**(최종에서만).
- **각 Wave 끝**: `npx tsc --noEmit` + `npm run build` 통과 → 변경 파일만 개별 커밋 → 멈추고 보고 → 승인 후 다음 Wave. **(bug-detective·code-guardian은 Wave마다 돌리지 말 것 — 최종에 한 번만.)**
- 각 보고: **①관찰 ②근거(파일:줄) ③공용 영향 범위 ④결정적 코드임.** 한국어로.
- 스키마 변경(컬럼 추가 등)은 **마이그레이션 파일만 작성**, 적용은 Steve가 SQL Editor에서 직접(CC는 DB 직접 적용 금지). 비파괴 ADD COLUMN·기본값 명시.

## WAVE 1 — 관리자 제품 등록/편집 화면 폭 넓히기 (빠름, 먼저)
- **문제**: 관리자 제품 등록/편집 폼(ProductForm 및 그 페이지 래퍼)이 메뉴 제외 영역의 **절반 폭만** 사용해 좁다. 넓게 쓰고 싶음.
- 해당 페이지의 컨테이너 **max-width 제약**(예: `max-w-md`·`max-w-xl`·`max-w-2xl` 등)이나 그리드/중앙정렬이 폭을 좁히는 지점을 찾아, 관리자 콘텐츠 영역(사이드 메뉴 제외한 전체)을 **넓게(예: max-w-4xl~full 또는 관리자 다른 넓은 페이지와 동일 기준)** 쓰도록 조정.
- 다른 관리자 페이지(주문·사용자 목록 등)가 이미 넓게 쓰는 기준이 있으면 **그 기준과 통일**(보편 해결). 필드가 많으면 2열 그리드로 배치해 넓은 폭 활용도 고려(단 과설계 말고 폭 확장 우선).
- ⚠️ 관리자 레이아웃(사이드바+콘텐츠) 공통 래퍼를 건드리면 다른 관리자 페이지에 영향 가니, **제품 폼 페이지에 국한**해서 조정. 공통 래퍼 수정이 불가피하면 영향 범위 확인 후 보고.
검증 → 변경 파일만 커밋 → **STOP & 보고.**
