# fix-stage6-downloads-mobile

You are a senior Next.js 15 (App Router) / TypeScript / Supabase / Tailwind v4 engineer on the **CoreZent** web codebase (멀티 상품 SaaS 스토어). 6단계 2종을 Wave별로 처리한다. **사전 점검(READ-ONLY audit) 불필요 — 바로 작업.** 단, 작업 중 애매하거나 위험한 판단이 필요하면 **멈추고 한국어로 보고**한다.

## 공통 STRICT RULES
- 코드는 네가 안다. 사용자는 코드 세부를 모르니, **막히거나 애매한 결정이 생기면 추측하지 말고 멈추고 한국어로 보고**하고 지시를 기다린다.
- **보편 해결 / 하드코딩 금지.** in-place 수정, 지정 범위만, 무관 리팩터 금지. 버전·URL 데이터는 DB에서(하드코딩 금지).
- **결제·라이선스·웹훅·slug 로직 불변.**
- **파일 업로드 인프라 만들지 말 것** — #31은 **URL 수동 입력 방식으로 확정**(설치파일은 외부에 있고, 관리자는 URL·버전·릴리스노트만 관리). Supabase Storage 업로드 신설 금지.
- **다른 작업자/다른 CC 세션 미커밋 변경 미접촉.** `git add .` 금지(변경 파일만 개별 add). **Wave 중 push 금지**(최종에서만).
- **각 Wave 끝**: `npx tsc --noEmit` + `npm run build` 통과 → 변경 파일만 개별 커밋 → 멈추고 보고 → 승인 후 다음 Wave. **(bug-detective·code-guardian은 Wave마다 돌리지 말 것 — 최종에 한 번만.)**
- 각 보고: **①관찰 ②근거(파일:줄) ③공용 영향 범위 ④결정적 코드임.** 한국어로.
- 스키마 변경이 필요하면 **마이그레이션 파일만 작성**, 적용은 Steve가 SQL Editor에서 직접(CC는 DB 직접 적용 금지). 비파괴·기본값 명시.

## WAVE 1 — 다운로드 버전 관리 (관리자, URL 수동 입력 방식)
- **목표**: 관리자가 제품별 설치파일의 **버전·다운로드 URL·릴리스노트·게시일**을 관리하고, 회원 "내 라이선스" 다운로드 버튼(2단계에서 만든 것)이 **항상 최신 버전**을 가리키게.
- 현재 구조 확인: changelog/릴리스 데이터에 download_urls가 이미 있으면 **그 구조를 확장**(새 테이블 신설보다 기존 재사용 우선). 없으면 최소 스키마(제품FK·version·download_url·release_note·released_at·is_latest 또는 정렬 규칙)로 마이그레이션 파일.
- 관리자 UI: 제품별 버전 목록(최신이 위) + 새 버전 추가 폼(버전·URL·릴리스노트) + 수정/비활성. **파일 업로드 없음 — URL 입력만.**
- 회원 쪽 연결: "내 라이선스" 다운로드 버튼과 (있으면) changelog 페이지가 이 데이터의 **최신 버전**을 읽게. OS별 URL이 필요한 구조면(윈도/맥) 필드 분리 — 현 제품이 단일 OS면 단순하게(과설계 금지).
- URL 유효성: 형식 검증 정도만(http/https). 외부 URL 접근성 체크까지는 불요.
검증 → 변경 파일만 커밋 → **STOP & 보고.**

## WAVE 2 — 모바일 화면 점검·보정 (공개 사이트 중심)
- **목표**: 모바일(대략 360~430px 폭)에서 공개 사이트가 깨지거나 불편한 곳을 찾아 보정. 회원 대시보드는 주요 화면(라이선스·설정)만 가볍게.
- 진행 방식: 페이지별로 모바일 뷰포트 기준 점검 → 문제 지점 수정. 대상: 랜딩(/)·/pricing·/product(+상세가 있으면)·about/contact/faq/changelog·Navbar(모바일 메뉴)·Footer·로그인/가입 → 대시보드 라이선스·설정.
- 흔한 문제 유형: 가로 스크롤 발생(고정폭·긴 단어), 겹침/잘림, 터치 타겟 너무 작음, 표가 화면을 넘음(→가로 스크롤 래퍼 또는 카드형), 폰트 과대/과소, 버튼이 화면 밖, 모바일 메뉴 동작.
- **수정 원칙**: Tailwind 반응형 유틸(sm:/md:) 위주 최소 수정. 레이아웃 대수술 금지 — 깨짐 보정이 목적. 디자인 변경이 필요해 보이는 큰 건은 목록으로 보고만(수정은 승인 후).
- 보고에 **페이지별 발견→수정 목록**(파일:줄) 포함. 스크린샷 불가 환경이면 코드 기준으로 판단하되, 확신 없는 시각적 판단은 "Steve 육안 확인 필요" 목록으로 분리.
검증 → 커밋 → **STOP & 보고.**
